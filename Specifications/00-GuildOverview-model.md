# Guild Overview v1 — Data Model

## Назначение

Документ описывает внутреннюю модель данных, используемую модулем Guild Overview.

Источник данных — Comlink API (`POST /guild`).

Важно:

- Guild Overview **не работает напрямую** с ответом Comlink API.
- Ответ Comlink один раз преобразуется во внутреннюю модель приложения.
- Внутренняя модель используется всеми компонентами Guild Overview.
- Поля, не используемые модулем, не включаются в модель.

---

# 1. Данные участников гильдии

## playerId

Источник:

```json
playerId
```

Использование:

- уникальный идентификатор игрока;
- используется для внутренних операций приложения;
- по умолчанию не отображается.

Интерфейс:

- отображение включается через настройки таблицы (кнопка ⚙).

---

## playerName

Источник:

```json
playerName
```

Использование:

- обязательное отображение в таблице.

---

## playerLevel

Источник:

```json
playerLevel
```

Использование:

- для большинства игровых гильдий всегда равно 85;
- полезно для молодых гильдий.

Интерфейс:

- отображается;
- может быть скрыт пользователем.

---

## memberLevel

Источник:

```json
memberLevel
"memberLevel": 4 - Leader
"memberLevel": 3 - Officer
"memberLevel": 2 - Member
```

Использование:

Определяет роль игрока в гильдии.

Интерфейс:

- отображается;
- может быть скрыт пользователем.

---

## lastActivityTime

Источник:

```json
lastActivityTime
```

Использование:

Обязательное поле.

Используется:

- отображение времени последней активности;
- сортировка игроков;
- последующий расчёт активности;
- модуль Territory War Planner.

---

## guildJoinTime

Источник:

```json
guildJoinTime
```

Использование:

Используется для вычисления:

- стажа игрока в гильдии;
- среднего количества ежедневных рейдовых купонов.

---

## galacticPower

Источник:

```json
galacticPower
```

Использование:

Используется:

- отображение общего ГМ игрока;
- вычисление возможности исключения неактивного игрока без потери доступа к Территориальным битвам.

---

## characterGalacticPower

Источник:

```json
characterGalacticPower
```

Использование:

Не используется Guild Overview v1.

Интерфейс:

- скрыт по умолчанию;
- может быть включён пользователем.

Используется позднее модулем TB Planner.

---

## shipGalacticPower

Источник:

```json
shipGalacticPower
```

Использование:

Не используется Guild Overview v1.

Интерфейс:

- скрыт по умолчанию;
- может быть включён пользователем.

Используется позднее модулем TB Planner.

---

# 2. Member Contribution

Источник:

```json
memberContribution
```

После преобразования Comlink JSON поле type больше нигде не используется.

Внутренняя модель:

```text
raidTickets
gearDonations
guildTokens
```

---

## raidTickets

Источник:

type = 2

Использование:

current

- отображается;
- используется при сортировке;
- используется для контроля ежедневной активности.

lifetime

- сохраняется;
- используется для статистики.

---

## gearDonations

Источник:

type = 3

Использование:

current

- по умолчанию скрыт;
- может быть включён пользователем.

lifetime

- сохраняется;
- может использоваться расширенной статистикой.

---

## guildTokens

Источник:

type = 1

Использование:

В Guild Overview v1 не используется.

Причина:

невозможно гарантировать, что lifetime относится только к текущей гильдии.

Поле сохраняется в модели, но не используется интерфейсом.

---

# 3. Последний рейд

Источник:

```json
lastRaidPointsSummary
```

Используются поля:

```text
campaignNodeId
totalPoints
```

---

## campaignNodeId

Название последнего завершённого рейда.

Используется для отображения.

---

## totalPoints

Максимальный счёт, набранный игроком в последнем завершённом рейде.

Используется для отображения и сортировки.

---

# 4. История Territory War

Источник:

```json
recentTerritoryWarResult
```

Используются:

```text
score
power
opponentScore
endTimeSeconds
```

---

## score

Счёт гильдии.

---

## power

Суммарная зарегистрированная галактическая мощь.

Используется для анализа активности игроков.

---

## opponentScore

Счёт противника.

---

## endTimeSeconds

Дата окончания войны.

Используется для построения истории.

---

# 5. Данные гильдии

Источник:

```json
profile
```

---

## id

Уникальный идентификатор гильдии.

---

## name

Название гильдии.

Обязательное отображение.

---

## externalMessageKey

Описание гильдии.

Может содержать ссылки.

По умолчанию не отображается.

---

## memberCount

Количество участников.

---

## memberMax

Максимальное количество участников.

---

## levelRequirement

Минимальный уровень вступления.

---

## guildGalacticPower

Общая галактическая мощь.

Используется для отображения.

---

## guildGalacticPowerForRequirement

Ключевой показатель.

Используется для расчёта возможности удаления неактивных игроков без потери доступа к Территориальным битвам (Rise of the Empire, Geonosis).

---

## guildEventTracker

Используется только последний завершённый TB.

Используются поля:

```text
definitionId
completedStars
```

---

### definitionId

Название последней Территориальной битвы.

---

### completedStars

Количество полученных звёзд.

Используется для отображения и последующего анализа эффективности гильдии.
